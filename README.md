# Comunalka AI

Telegram-бот для первичного анализа квитанций ЖКХ по фото: OCR, извлечение данных и сверка начислений с базовыми тарифами.

Проект находится на стадии `MVP` и развивается как личный pet-проект с упором на продуктовый и инженерный рост.

## Что уже реализовано

- Прием квитанций в Telegram (фото и файл-изображение).
- Предобработка изображения (повороты, авто-кроп, усиление контраста).
- OCR через Yandex Vision.
- Извлечение структурированных полей через YandexGPT.
- Проверка начислений и расчёт потенциальных расхождений.

## Технологии

- Python 3.11+
- aiogram 3
- aiohttp
- Pillow
- python-dotenv

## Структура проекта

```text
bot/
  core/        # OCR, LLM-парсинг, расчеты и конфиг
  handlers/    # Telegram-хэндлеры
  main.py      # точка входа
```

## Локальный запуск

1) Клонировать репозиторий и перейти в каталог `bot`:

```bash
cd bot
```

2) Создать и активировать виртуальное окружение:

```bash
python -m venv .venv
# Windows PowerShell
.venv\Scripts\Activate.ps1
```

3) Установить зависимости:

```bash
pip install -r requirements.txt
```

4) Создать `.env` на основе примера:

```bash
copy .env.example .env
```

5) Заполнить переменные:

- `BOT_TOKEN`
- `YANDEX_FOLDER_ID`
- `YANDEX_API_KEY`

6) Запустить бота:

```bash
python main.py
```

## Тесты


## Ограничения текущего MVP

- Тарифы пока захардкожены и не привязаны к дате/региону.
- Качество извлечения зависит от OCR качества исходного фото.
- Проверка не является юридическим заключением, а дает ориентир для ручной валидации.

## Roadmap

- Перенос тарифов в управляемый источник (БД/конфиг по регионам).
- Улучшение устойчивости парсинга на «грязном» OCR.
- Расширение аналитики (помесячная динамика, детальные правила аномалий).
- Добавление автоматических тестов для OCR/LLM пайплайна.

## Этот проект демонстрирует:

- работу с асинхронным Python и Telegram API;
- интеграцию внешних AI API (OCR + LLM);
- проектирование пайплайна обработки неструктурированных данных;
- базовую инженерную дисциплину (конфиги, тесты, документация).
